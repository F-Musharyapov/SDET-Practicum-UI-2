# Тест-кейсы

## ID: 1
## Успешное создание клиента
### Приоритет: High (Высокий)

## Шаги:
1. Перейти на страницу https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager.
2. Нажать кнопку Add Customer.
3. Сгенерировать Post Code из 10 цифр (Пример: 0001252667)
4. Сгенерировать First Name, на основе Post Code, разбив условно на 5 двузначных чисел и преобразовав в букву английского алфавита по порядку от 0 до 25 (Пример: abzap)
5. Заполнить форму:
- В поле "First Name" ввести сгенерированное имя: xbuyc
- В поле "Last Name" ввести валидное имя: например Dumbledore
- В поле "Post Code" ввести сгенерированный код: 0001252667
6. Нажать кнопку Add Customer

## Ожидаемый результат: 
Выводится всплывающее окно с сообщением “Customer added successfully with customer id :{id}”.


## ID: 2
## Сортировка клиентов по убыванию First Name
### Приоритет: Medium (Средний)
## Предусловие: 
Таблица Customers содержит несколько строк с различными значениями поля First Name

## Шаги:
1. Перейти на страницу https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager.
2. Нажать кнопку Customers.
3. В табличной части нажать на заголовок первого поля “First Name” для сортировки по убыванию и зафиксировать отображение
4. Кликнуть на заголовок еще раз для сортировки по возрастанию и зафиксировать отображение

## Ожидаемый результат: 
1. При сортировке по убыванию имена отображаются в обратном алфавитном порядке (Z-A)
2. При сортировке по возрастанию имена отображаются в алфавитном порядке (A-Z)

## ID: 3
## Сортировка клиентов по возрастанию First Name
### Приоритет: Middle (Средний)
## Предусловие: 
Таблица Customers содержит несколько строк с различными значениями поля First Name

## Шаги:
1. Перейти на страницу https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager.
2. Нажать кнопку Customers.
3. В табличной части нажать на заголовок первого поля “First Name”.
4. Повторно в табличной части нажать на заголовок первого поля “First Name”.

## Ожидаемый результат: 
Данные в табличной части сортируются по возрастанию данных в поле First Name.


## ID: 4
## Успешное удаление клиента, длина которого ближе всего к среднему арифметическому длин всех имен
### Приоритет: High (Высокий)
## Предусловие: 
Таблица Customers содержит минимум 3 строки с различными значениями поля First Name

## Шаги:
1. Перейти на страницу https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager.
2. Нажать кнопку Customers.
3. Извлечь из таблицы Customers все значения First Name
4. Рассчитать среднее арифметическое длины имен
5. Опеределить Customer с длиной First Name, наиболее близкой к средне арифметической длины всех имен
3. Нажать кнопку Delete для Customer с именем, длина которого ближе к среднему арифметическому длин всех First Name клиентов.

## Ожидаемый результат: 
Успешное удаление из табличной части соответствующей строки с данными Customer.
